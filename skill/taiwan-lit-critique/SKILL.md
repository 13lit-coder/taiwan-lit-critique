# 台灣文學文本批判 SOP

把 Claude Code 當成一個**研究工作台**，而不是一台「給我答案的機器」。核心信念：
研究生層級的批判思考有兩層——**批判文本**，以及**批判 AI 對文本的詮釋**。第二層才是重點。
這個 skill 的每一步都要為這兩層服務。

## 適用時機

當使用者要：分析台灣文學文本、做細讀、套用文學理論、做詞頻/語料的量化分析、
帶研究生做文本批判、出討論題或評分標準時。若使用者只是想「快速摘要一篇小說」，
不要硬套完整流程——先問他要的是理解還是批判。

---

## 開始前：確認研究設定（30 秒，不要跳過）

批判閱讀的品質取決於「先講清楚立場」。動手前先確認三件事（缺哪個就問使用者）：

1. **文本在哪** — 語料檔案的路徑。若還沒有語料庫，建議這個結構：
   ```
   corpus/<作者>/<篇名>.txt      ← 文本（純文字或 markdown）
   readings/                     ← 二手研究、理論選文
   analyses/<篇名>-<日期>.md      ← 分析成果
   ```
   純文字檔要能被 Read + Grep，才能引用行號當證據。

2. **批判視角** — 要用哪個理論透鏡？（後殖民 / 國族認同 / 性別 / 階級 / 精神分析 / 
   形式主義 / 生態批評…）不確定就先做無預設的細讀，再回頭選透鏡。透鏡細節見
   `references/theory-lenses.md`。

3. **產出形式** — 是研究札記、討論題、評分 rubric、還是量化圖表？

如果專案根目錄還沒有 `CLAUDE.md`，建議寫一份，把上述立場固定下來，讓每次分析都在同一批判規範下進行。

---

## 三條不可退讓的證據紀律

這是整個 skill 的地基。AI 最大的風險是「講得頭頭是道但沒有文本根據」，所以：

1. **每個詮釋都要附文本證據，標到行號**（`檔名:行號`）。Claude Code 能引用行號，
   這是它相對於一般聊天視窗最大的研究優勢——請善用，並讓使用者可以當場查證。

2. **嚴格區分三種陳述**，在札記裡用不同標記分開：
   - 📄 **文本事實**：文本字面上寫了什麼（可被引文驗證）
   - 🔍 **詮釋推論**：從文本推出的解讀（要說明推論鏈）
   - 🎓 **理論套用**：借外部理論框架的閱讀（要自覺框架的預設與限制）
   
   混淆這三者，是研究生文本分析最常見的失分點。分開標記能讓思考變可見。

3. **拒絕無證據的概括**。遇到「這反映了台灣人的集體焦慮」這種大詞，就要回頭問：
   哪一段文本支持？還是這只是套版話術？寧可說「證據不足」也不要編。

## 工作流程：選一個或組合多個模式

### 模式 A ── 細讀（Close Reading）

不預設結論，逐段拆解。對指定段落依序檢視：

1. **語言與修辭**：反諷、隱喻、重複、語域（正式/口語/方言）、命名。
2. **敘事**：視角（誰在看/誰在說）、時間處理、聚焦、敘事者可信度。
3. **結構與張力**：對比、留白、未言明之物（silence 常比明說更有分析價值）。
4. **多語現象**：台語、日語、華語的交雜與其政治意涵（台灣文學特有，見脈絡檔）。

每一觀察都附 `檔名:行號`。輸出時用下方「札記模板」。

### 模式 B ── 理論透鏡對讀

選 2–3 個透鏡，各讀一遍**同一段文本**，再刻意找出它們之間的張力或矛盾。
目的不是找到「正確答案」，而是讓學生看見**詮釋是被框架決定的**——這本身就是批判自覺的核心訓練。
各透鏡的關鍵提問、代表理論家、以及常見誤用陷阱，讀 `references/theory-lenses.md`。

### 模式 C ── 遠讀（Distant Reading / 數位人文）

跨文本、跨年代的量化證據。適合的問題：某關鍵詞（如「祖國」「本島人」「內地」「番」）
在不同時代文本的分布、作家風格比較、主題詞的共現。用小程式跑，結果**要回扣細讀**，
不能只丟數字。具體 recipe（詞頻、關鍵詞脈絡 KWIC、TF-IDF、斷詞注意事項）見
`references/distant-reading.md`。

### 模式 D ── 批判 AI 的詮釋 ⭐ 核心，別省略

前三個模式做完後（或針對任何 AI 產出的詮釋），一定要做這一層。這是把「用 AI」
升級成「研究方法」的關鍵：

1. **自我反駁**：對剛才給出的每個主要詮釋，主動提出一個有力的反面讀法。
2. **抓幻覺**：檢查所有引文是否真的存在於文本（用 Grep/Read 逐條核對行號）。
   明確標出任何「看似合理但查無此段」的地方。
3. **抓套版**：標出哪些論點「換成任何一篇小說都成立」——那通常是空話，不是分析。
4. **補在地脈絡**：AI 對台灣在地脈絡（二二八、白色恐怖、鄉土文學論戰、台語書寫政治、
   日治語言政策）常常淺薄或失準。主動標記「此處需人工查證/補充脈絡」，並邀請使用者糾正。

這一層的產出，寫成札記裡的「與 AI 論辯」段落——研究生繳交時這往往比詮釋本身更有價值。

## 台灣文學脈絡（動手前速查）

台灣文學有很多 AI 容易出錯的在地知識：文學分期、多語書寫、關鍵歷史事件、
作家的政治位置。**在做任何時代脈絡的斷言前**，先讀 `references/taiwan-lit-context.md`，
並對戰後仍有版權的作家提醒使用者只做節錄/合理使用。

---

## 輸出：分析札記模板

除非使用者另有指定，分析成果一律用這個結構，存到 `analyses/<篇名>-<日期>.md`：

```markdown
# <篇名>（<作者>，<年代>）批判閱讀札記

## 一、文本定位
- 版本/出處、篇幅、寫作與發表年代、關鍵脈絡（一兩句）

## 二、細讀觀察
（每點標記 📄 事實 / 🔍 推論 / 🎓 理論，並附 `檔名:行號`）

## 三、理論透鏡（若採用）
- 透鏡 A 的讀法 →
- 透鏡 B 的讀法 →
- 兩者張力 →

## 四、遠讀證據（若採用）
- 量化發現 + 回扣文本的解釋

## 五、與 AI 論辯（必寫）
- 反面讀法：
- 查證結果（引文真偽、行號核對）：
- 套版話術檢討：
- 需人工補充的在地脈絡：

## 六、可延伸的研究問題
- （給研究生繼續往下做的 3 個問題）
```

## 收尾

- 把札記存進 `analyses/`。
- 如果專案是 git repo，建議 commit——git log 會變成學生的**詮釋演進史**，
  本身就是很好的思考歷程記錄（若使用者要求再 commit，不要擅自 push）。
